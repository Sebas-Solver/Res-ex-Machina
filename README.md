## 🧬 What is Res ex Machina?

**Res ex Machina** is a platform for **technical registration of AI generation events**. It doesn't judge, validate, or moderate. It only records and cryptographically anchors.

When an AI agent generates an output (text, image, code, audio...), it can register that event in Res ex Machina with a signed **Proof of Generation (PoG)**. The system:

1. ✅ Verifies the agent's cryptographic signature (EIP-712)
2. ✅ Records the event with an immutable timestamp
3. ✅ Verifies the on-chain fee payment
4. ✅ Anchors the record on an L2 EVM blockchain
5. ✅ Returns a receipt verifiable by third parties

> **"The thing born of the machine"** — A registry that doesn't say whether something is good, legal, or original. It only says: *this was declared as generated by this technical identity, at this moment*.

## 🎯 Problem It Solves

| Current Problem | Res ex Machina Solution |
|---|---|
| AI outputs have no verifiable traceability | Immutable record with cryptographic signature |
| No **positive** registration system for AI exists | PoG: the agent proactively declares what it generates |
| Agents generate without leaving verifiable memory | Verifiable timeline by hash or record ID |
| Regulatory compliance (EU AI Act) requires traceability | Neutral, public, and auditable registry |

## 🏗️ Foundational Principles

```
NEUTRALITY       → Does not judge content. Records facts.
IMMUTABILITY     → What's recorded is never deleted or modified.
AUTOMATION       → Designed for agents, not for humans.
VERIFIABILITY    → Everything is cryptographically verifiable.
NON-CUSTODIAL    → The agent controls its identity (wallet).
```

## 📦 What Gets Recorded (and What Doesn't)

### ✅ Always Recorded
- `content_hash` — SHA-256 of the generated output
- `agent_wallet` — Agent's cryptographic identity
- `timestamp` — Declared moment of generation
- `pog_bundle` — Proof of Generation v1, signed (EIP-712)
- `generation_process` — Type, level of human intervention, pipeline
- On-chain anchoring status

### ❌ Never Recorded (in v1)
- The content itself (only the hash)
- AI detection / originality scoring
- Any kind of human validation
- Semantic analysis or moderation
- Copyright claims

## 🛠️ Architecture (v1)

```
AI Agent ──────────── REST API ──────────── PostgreSQL
  (wallet)    EIP-712     │                    (records)
                          │
                     Redis (queue)
```

### Endpoints

| Method | Path | Auth | Description |
|---|---|---|---|
| `GET` | `/v1/health` | — | System status (DB, Redis, L2) |
| `POST` | `/v1/records` | Wallet (EIP-712) | Register a generation event |
| `POST` | `/v1/records?wait_for_anchor=true` | Wallet (EIP-712) | Create + wait for anchoring (max 25s) |
| `GET` | `/v1/records/{id}` | — | Query by ID |
| `GET` | `/v1/records/verify?content_hash=` | — | Verify by hash |
| `GET` | `/v1/records` | — | List records by wallet (filters, pagination, sort) |
| `GET` | `/v1/records/mine` | Wallet (EIP-191) | List agent's own records |
| `GET` | `/v1/records/{id}/export` | — | Export verifiable receipt |
| `GET` | `/v1/records/{id}/export?mode=compact` | — | Compact receipt (verification only) |
| `POST` | `/v1/records/batch` | Wallet (EIP-712) | Create up to 100 records in a single call |
| `POST` | `/v1/webhooks` | Wallet (EIP-191) | Register notification webhook |
| `GET` | `/v1/webhooks` | Wallet (EIP-191) | List own webhooks |
| `DELETE` | `/v1/webhooks/{id}` | Wallet (EIP-191) | Deactivate webhook |

### Anti-Abuse Measures
- **On-chain fee** required — verified with 5 checks (exists, confirmed, amount, recipient, recent)
- **Rate limiting** — 100 req/min global, 10 req/min POST /records
- **Idempotency** — content_hash unique (409)
- **Unique nonce** — per wallet, anti-replay (409)
- **Body limit** — 64KB maximum
- **Helmet** — Automatic security headers
- **SSRF** — Webhooks HTTPS-only, private/localhost IPs blocked
- **HMAC-SHA256** — Webhook payload signing

Trivial integration — from 50+ lines of code to 5:

```typescript
import { RxMClient } from '@res-ex-machina/sdk';

const rxm = new RxMClient({ account: wallet, rpcUrl: '...', apiUrl: '...' });
const receipt = await rxm.record('AI generated content', {
  modelId: 'openai:gpt-4o:2026-01',
  contentType: 'text/plain',
});
```

**Features**: WebCrypto hashing, EIP-712 signing, automatic fee payment (or BYO), retry with backoff, webhooks subclient, typed errors.

## 🔐 Security by Design

The system has **24 invariants** that are never violated:

- **INV-001**: Records are permanent — cannot be deleted (DELETE → 405)
- **INV-005**: No human validates or invalidates a record
- **INV-007**: The platform does NOT custody private keys
- **INV-009**: Every PoG must be signed by the declared agent
- **INV-012**: No record without paid fee
- **INV-022**: No scoring, rankings, or reliability labels

### Implemented Measures

| Measure | Implementation |
|---|---|
| Security headers | `@fastify/helmet` |
| CORS | `@fastify/cors` (disabled in prod) |
| Rate limiting | `@fastify/rate-limit` per IP |
| Body limit | 64KB maximum |
| Error sanitization | Never exposes stack traces |
| Digital signatures | EIP-712 (verifyTypedData) |
| Fee verification | 5 on-chain checks |
| Idempotency | UNIQUE constraints in DB |

## 🤝 Project Philosophy

> *"The system provides data; interpretation is the user's responsibility."*

Res ex Machina **does not make judgments**. It doesn't say whether something is original, good, legal, or valuable. It is a **technical registry**: it records signed declarations and anchors them in time.

This is deliberate. In a world where AI generation is increasingly ubiquitous, what's missing is not a judge, but a **neutral registry of facts**.
